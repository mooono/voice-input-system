# 推敲応答速度最適化 設計書

## 問題

音声認識後の推敲（LLM呼び出し）に約1.5秒かかっており、体感速度が遅い。
目標: **0.6〜0.8秒以内**（一括LLM呼び出しを維持）。

## アプローチ

現行のアーキテクチャ（録音停止 → 一括LLM推敲 → ペースト）を維持しつつ、
不要な処理の除去・前処理の追加・設定チューニングで高速化する。

## 変更一覧

### 1. ウィンドウ再アクティベーション除去（-300〜500ms）

バックグラウンドでホットキーを監視するため、ペースト先ウィンドウの
再アクティベーションは不要。アクセシビリティ権限設定済みで正常動作する。

- `main.py`: `get_frontmost_app()` 呼び出しと `target_app` 受け渡しを除去
- `output.py`: `paste_text()` から `target_app` パラメータを除去、
  `_activate_app()` と `get_frontmost_app()` 関数を削除

### 2. Azure STT TrueText有効化

`recognizer.py` で `SpeechConfig` に postprocessing オプションを設定し、
STT側で句読点・大文字正規化・数値変換を行う。LLMが処理すべき整形量を削減。

### 3. システムプロンプト最適化（-20〜50ms）

現在98文字のプロンプトを短縮しつつ、フィラーワード除去（えーっと、あの等）の
指示を追加。ルールベースではなくLLMに任せることで、想定外のフィラーにも対応。

### 4. ポストホットキー遅延短縮（-150ms）

`.env` の `OUTPUT_POST_HOTKEY_DELAY_SEC` を `0.25` → `0.1` に変更。
LLM処理時間中にキー状態は安定するため、長い遅延は不要。

## 影響範囲

- `src/refiner.py` — システムプロンプト最適化（フィラー除去指示追加）
- `src/recognizer.py` — TrueText設定追加
- `src/main.py` — ウィンドウ管理除去
- `src/output.py` — target_app関連コード除去
- `.env` — OUTPUT_POST_HOTKEY_DELAY_SEC 変更
